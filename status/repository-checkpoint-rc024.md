## Phase Transition

The most important event between RC-020 and RC-024 is not a count movement. It is a shift in operating mode.

**RC-001 through RC-020 — Cycle 1 and 2:** Observation → Recurrence → Count

**RC-021 through RC-024 — Cycle 3:** Observation → Candidate Explanation → Adversarial Test → Revision

RC-021 (CRISPR) extended Watch B's suppression pattern beyond the policy domain.
RC-022 (CCTs) confirmed pilot-to-policy at three instances and Watch B suppression at five.
RC-023 (ORT) attacked Watch A's implied association between pilot-to-policy structure and transferability failure. The attack succeeded: one pilot-to-policy case earned Supported status with transferability absent.
RC-024 (PEPFAR) attacked the prerequisite-universality explanation that emerged from RC-023. The attack succeeded in a different direction: prerequisites were context-specific and non-universal, yet transfer succeeded. The explanatory variable had to move.

The repository is recording this transition because Cycle 3's epistemic state is not derivable from the count table alone. AGRA (RC-025) will immediately begin modifying the explanatory layer. This checkpoint preserves what the layer looked like before that perturbation.

### Prerequisite state taxonomy (new at Cycle 3)

Three states, reached through different routes:

| State | Description | Examples |
|---|---|---|
| Already Present | Prerequisites near-universal; present in essentially all transfer contexts without construction | RC-023 (water, salt, sugar for ORT) |
| Deliberately Built | Prerequisites context-specific; absent initially; constructed through deliberate resource mobilization before or during transfer | RC-024 (drug supply, monitoring, trained providers for HIV ART via PEPFAR) |
| Not Yet Available | Prerequisites context-specific; absent; not yet constructed at transfer | RC-017, RC-018, RC-022 (institutional complements for CCT, UBI, housing mobility) |

The operative word is **available** — a state, not a property. Availability can be reached through multiple paths. The taxonomy's predictive claim: prerequisites available at transfer → transfer succeeds; prerequisites unavailable at transfer → transfer uncertain.

This taxonomy is not validated. It is derived from seven cases (2 supported, 3 unresolved, 1 mixed, 1 transferability-absent-with-uncertain-status). RC-025 will pressure it directly.

## Layer 3: Candidate Explanations

### CE-1 — Source-context identification quality predicts Watch B likelihood

Evidence: 5 instances of strong ID (RCT, lottery RCT, quasi-experimental, clinical trials, FDA approval) — Watch B absent in all 5. 1 instance of weak/observational ID (RC-019) — Watch B present.

Status: Emerging. 6 instances. One disconfirmation would require: strong source-context ID with Watch B present, or weak ID with Watch B absent.

Not promoted. Not discarded.

### CE-2 — Prerequisite availability at transfer predicts transfer outcome

Evidence: 2 instances of prerequisites available (RC-023, RC-024) — transfer succeeded in both. 3 instances of prerequisites unavailable (RC-017, RC-018, RC-022) — transfer uncertain in all 3. 1 instance mixed (RC-022) — outcome mixed.

Status: Emerging. Pattern consistent but derived from small sample and two-audit run of Supported results.

Critical unresolved question: is availability the decisive variable, or is availability downstream of something else (resources, time, structural buildability)?

RC-023 and RC-024 now illustrate two different paths to the same state. That is evidence for availability as the operative variable. But it does not establish why availability materializes in some cases and not others.

## Layer 4: Competing Explanations for "Not Yet Available"

The primary unresolved frontier is not the watch items. It is this:

Why does availability emerge in some contexts and not others?

Three competing models, not yet distinguishable from the current evidence base:

**Model A — Resource Constraint:** Prerequisites can be built but funding is absent. Prediction: provide sufficient resources → prerequisites build → transfer succeeds. PEPFAR appears consistent with this model. But PEPFAR had exceptional resource mobilization ($85B, 20 years, US diplomatic leverage). Whether "sufficient resources" is a realistic lever for most transfer contexts is untested.

**Model B — Time Constraint:** Prerequisites can be built but not within the policy decision horizon. Transfer fails not because prerequisites are unachievable but because the required construction period exceeds the implementation timeline. Prediction: interventions with longer institutionalization phases will eventually transfer; interventions assessed on short timescales will appear to fail.

**Model C — Structural Non-Buildability:** Some prerequisites cannot realistically be built regardless of resources or time. Cultural trust, political legitimacy, social cohesion, and deep institutional norms may fall into this category — qualitatively different from laboratories, drug supply chains, or midwifery training. Prediction: high-resource, long-horizon transfer attempts targeting structurally non-buildable prerequisites will still fail.

These models make different predictions. RC-025 (AGRA) is selected to discriminate among them:

- If AGRA with substantial funding still produced weak transfer: Model A is weakened. Something other than resources is constraining availability.
- If AGRA produced partial success correlated with prerequisite construction: the taxonomy produces a graded prediction that fits.
- If AGRA succeeded: the current Unresolved cases (RC-017, RC-018, RC-022) may be resource-or-time constrained, not structurally limited.

## Summary

The repository entered Cycle 3 with two Watch A structural pairs and Watch B as its strongest pattern.

It exits Cycle 3 with:

1. Watch A's pilot-to-policy structure now having two successful transfer instances alongside three failures — the structure no longer implies transferability failure
2. The prerequisite availability taxonomy as the leading explanatory structure
3. Two candidate explanations (CE-1, CE-2) in emerging but unvalidated status
4. Three competing models (A, B, C) for the mechanism governing availability emergence
5. A defined adversarial sequence: RC-025 (AGRA) to pressure resources-versus-buildability; RC-026 (Sri Lanka maternal mortality) to attack from the opposite side

The repository is now in a phase where the highest-value audits are those designed to discriminate among explanations, not to add confirming instances. That is a different selection criterion than was operative in Cycles 1 and 2.

---

## Checkpoint Conclusion

The decisive insight of Cycle 3 is not that watch counts grew.

It is that the same outcome (transfer success or failure) can now be traced to a candidate causal variable (prerequisite availability), and that variable has two known paths to the "available" state (already-present, deliberately-built) and one known path to the "unavailable" state (not-yet-built).

The question Cycle 4 inherits is: what determines which path a given intervention takes?
